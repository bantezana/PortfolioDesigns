Brand Identity Kit

The visual system behind this portfolio

A warm, editorial identity built around one idea: design for the overlooked shouldn't look cold. Paper tones, serif warmth, and a single accent color per project keep the system consistent while giving each case study its own voice.

1 - Logo / Wordmark

Wordmark

No icon, no lockup gymnastics, the name itself is the mark, set in Fraunces to signal editorial craft over corporate polish. A monogram exists for tight spaces (favicons, social avatars) using the shared accent color.

3 - Typography

Type system

Fraunces carries the voice, warm, slightly literary, used for anything that needs presence. Inter handles everything that needs to be read quickly and clearly: body copy, labels, UI chrome.

All dividers use a 1px hairline in Line (#DDD6C7), never a heavy rule.

Grid & containers

Marketing pages (Home, Brand) use a 1180px container. Reading-heavy pages (Case Studies, About, Contact) narrow to 760–860px to keep line length comfortable.

Alignment

Left-aligned throughout, no centered blocks of text. Numbers and stats align in flexible rows so they reflow predictably on smaller screens.

5 - Imagery & Mood

Direction

Every visual choice should feel considered rather than decorative, closer to a print magazine than a SaaS landing page.

  • Warm, not clinical

    Paper backgrounds and ink text instead of pure white/black, softer, more human, less "app store."

  • Editorial pacing

    Generous whitespace and hairline dividers let content breathe the way a well-set article does.

  • Product screenshots as evidence

    Real UI captures over stock photography or illustration the work speaks for itself.

  • One accent, one story

    Each case study's accent color is used sparingly for emphasis, not decoration so it reads as intentional.

6 - Mobile Considerations

Responsive behavior

The system is designed desktop-first visually, but every rule has a mobile counterpart rather than an afterthought breakpoint.

  • Fluid type

    Display headings use clamp() so they scale continuously instead of jumping at breakpoints.

  • Single-column collapse

    Two-column layouts (project rows, bio + sidebar, color grids) stack to one column under 800px.

  • Compact navigation

    Nav link spacing and type size reduce under 480px so all four links stay on one line without crowding.

  • Full-bleed media

    Project and screenshot images expand to full container width on mobile instead of staying fixed-size.
